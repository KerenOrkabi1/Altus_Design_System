# Operational Overview Family

## Purpose
Defines the reusable overview family category for operational, persona-led screens.

## Family summary
This family is used when a page needs to help a persona understand the current state, identify what needs attention, and use AI assistance to prioritize or act.

## Core anatomy
- Main canvas overview area
- Executive/state summary area
- Bucket-based or equivalent action-focused area
- Alternate list/table views where appropriate
- Visible AI panel with Ask Altus when this family is active on supported screen sizes

## Family flow
- Land on overview
- Scan current state
- Review items needing attention
- Use AI for summary, prioritization, or next actions
- Drill into an item when more detail is needed
- Return to the overview with updated state

## Placement rules
### Landing placement
- Top statement/summary always visible
- Visible AI panel plus Ask Altus
- Welcome/orientation allowed as part of the shell
- Broader summary visualizations may be visible by default

### Product or sub-level placement
- Same family structure and behavior
- Summary is collapsed by default to keep focus
- Buckets become more specific to that level
- User focuses on the active domain area

## Views
- Overview family can support bucket/board, list, and table views.
- The default view may vary by initiative and persona.
- View behavior should reference the approved global view patterns.

## Responsive variants
- Desktop/tablet may use the main overview layout.
- Mobile can use a different layout pattern while staying within the same family purpose.
