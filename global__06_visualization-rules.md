# Visualization Rules

## Purpose
Defines reusable visual logic for state, summary, trend, risk, and operational presentation.

## Principles
- Show, do not decorate.
- Summary first, then drill-down.
- Use charts only when they help action or understanding.
- Keep enterprise visual calm.

## Reusable visual patterns
- Executive summary/state block
- Health and trend indicators
- Urgency/risk cues
- Bucket or board visuals
- List and table presentation

## Rules
- Reusable visual patterns should reference approved global components.
- If a visual pattern changes, all screens using that pattern should inherit the latest approved version.
- Semantic states must be consistent across pages.
