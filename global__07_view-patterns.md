# View Patterns

## Purpose
Defines reusable view patterns that page families can reference.

## View types
- Bucket or board view
- List view
- Table view

## Rules
- Different pages may choose different default views.
- The same underlying data model should support view switching where appropriate.
- If a view pattern is redesigned and approved, the global reference should point to the latest version.

## Usage
- Overview family can use bucket, list, and table views.
- Future families may also reference these patterns.
